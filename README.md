# industrial-tourist-passport — Цифровой паспорт промышленного туриста

## Обзор проекта

**industrial-tourist-passport** — расширенный MVP веб-модуля для профориентационного промышленного туризма по техническому заданию команды 7. Приложение помогает школьникам 9-11 классов и студентам изучать предприятия Кировской области, проходить профориентационный тест, выбирать экскурсии и сохранять результат в цифровом паспорте.

В репозитории используется нормализованная публичная база из файла `Predpriatia_2025_1.xlsx`: 51 предприятие с отраслью, адресом, сайтом, описанием и данными экскурсионной программы. Персональные телефоны ответственных в публичные данные не включены.

Архитектура:

- **Frontend**: React 18 + Vite + Tailwind CSS
- **Backend**: Express 4
- **Данные**: SQLite 3 через `sqlite3`, без ORM и без `better-sqlite3`
- **Демо-режим**: `preview.html` и `industrial-tourist-passport-repo-single.html` открываются без установки зависимостей

## Функционал

### Для школьника или студента

- регистрация туриста: ФИО, e-mail, телефон, фотография;
- пошаговый профориентационный тест на 12 вопросов с возвратом к предыдущему шагу;
- основная и запасная профессиональная рекомендация;
- подбор профиля: технический, творческий или научно-производственный;
- каталог из 51 предприятия с поиском и фильтрами по отрасли и городу;
- отдельная вкладка с новостями предприятий;
- расписание на ближайшие три месяца с профильными фильтрами;
- запись на экскурсию с проверкой мест, времени до начала и пересечений;
- отмена активной записи не позднее чем за 24 часа;
- подтверждение записи с адресом, датой, временем и требованиями ТБ;
- обратная связь после экскурсии: напоминание вернуться в веб-приложение, переход к Яндекс-анкете, оценка 1-5 и фиксация впечатлений;
- цифровой паспорт с результатом теста, активными записями, достижениями и отзывом.

### Для команды проекта

- ручное ведение базы предприятий на этапе MVP;
- публикация новостей предприятий;
- настройка расписания экскурсий;
- просмотр записей и обратной связи;
- подготовка маршрутов и текстовых шаблонов.

### Для предприятия в будущем

- личный кабинет предприятия;
- самостоятельная публикация новостей;
- самостоятельное управление расписанием;
- загрузка маршрутов;
- согласование требований по охране труда.

## Роли пользователей и сценарии

| Роль | Описание | Основные сценарии |
|------|----------|-------------------|
| `tourist` | Школьник, студент или промышленный турист | Регистрируется, проходит тест, читает новости, выбирает экскурсию, получает паспорт, оставляет отзыв |
| `project_team` | Команда проекта | Заполняет предприятия, новости, расписание и проверяет обратную связь |
| `enterprise_admin` | Представитель предприятия, будущая роль | Ведёт карточку предприятия, новости и расписание |
| `platform_owner` | Владелец внешней платформы | Видит, как модуль может быть встроен в Госуслуги, Сферум или региональный портал |

## Основной сценарий MVP

1. Пользователь открывает страницу модуля внутри демо-платформы.
2. Создаёт аккаунт.
3. Проходит профориентационный тест из 12 вопросов.
4. Система определяет основное и запасное направление.
5. Пользователь смотрит предприятия и новости.
6. Переходит во вкладку «Расписание» с уже выбранным профильным фильтром.
7. Выбирает экскурсию и нажимает «Записаться».
8. Система создаёт подтверждение записи.
9. В цифровом паспорте появляется предприятие, адрес, дата, время, комментарий и требования ТБ.
10. После экскурсии пользователь возвращается во вкладку «Обратная связь».
11. Открывает Яндекс-анкету, ставит оценку и сохраняет впечатления.
12. Отзыв появляется в цифровом паспорте.

## API-маршруты

Базовый префикс: `/api`

### Служебные

- `GET /health` — проверка API.
- `GET /platform` — информация о демо-платформе.

### Каталог

- `GET /enterprises` — список предприятий.
- `GET /news` — новости предприятий.
- `GET /excursions` — расписание экскурсий.

### Пользователь

- `POST /auth/register` — регистрация.
- `POST /profile/photo` — загрузка фото.

### Профориентация

- `POST /career-test` — принять ответы и вернуть специальность.

### Записи

- `POST /bookings` — создать запись.
- `GET /bookings` — список записей.
- `DELETE /bookings/{id}` — отменить запись не позднее чем за 24 часа.

### Обратная связь

- `POST /feedback` — сохранить оценку, впечатления и отметку о Яндекс-анкете.
- `GET /feedback` — получить отзывы для команды проекта.

## LLM-интеграция

LLM не обязательна для MVP, но архитектура оставляет место для будущей генерации документов:

- анализ текстового маршрута;
- выделение опасных зон;
- генерация чек-листа техники безопасности;
- подготовка комментария предприятия;
- создание шаблонов документов.

API-ключи не должны попадать во frontend и GitHub. В репозитории хранится только `.env.example`.

## Запуск и настройка

### Полный режим

```bash
npm run install:all
npm run dev:server
npm run dev:client
```

Клиент: `http://localhost:5173`
API: `http://localhost:4000/api`

### Быстрый просмотр

Можно открыть `preview.html` или `industrial-tourist-passport-repo-single.html` напрямую в браузере. Эти файлы работают без npm и без сервера.

### Docker

```bash
cp server/.env.example server/.env
docker compose build
docker compose up -d
```

После запуска: `http://localhost:8080`

## Что не входит в первую версию

- автоматическая генерация 3D-карты маршрута;
- интеграция с видеонаблюдением предприятия;
- мобильный экскурсовод в реальном времени;
- электронная подпись документов;
- оплата через сайт;
- реальная интеграция с Госуслугами или Сферумом;
- полноценный личный кабинет предприятия.

## Проверка основного сценария

1. Открыть приложение.
2. Найти предприятие по названию и применить фильтр отрасли.
3. Зарегистрироваться.
4. Пройти пошаговый тест из 12 вопросов.
5. Перейти к экскурсиям, связанным с результатом.
6. Проверить состояние «Мест нет» и выбрать доступную экскурсию.
7. Проверить подтверждение и достижение в паспорте.
8. Отменить тестовую запись или перейти в «Обратную связь».
9. Открыть Яндекс-анкету, поставить оценку и сохранить впечатления.

---

## Критерий успешности MVP

Проект считается успешным, если через систему можно подготовить и показать не менее 10 экскурсий за 1 месяц вместо ручной подготовки, а пользователь проходит полный путь от регистрации до подтверждения записи и оставляет обратную связь после посещения.
